# Projects & Environments

---

## Creating a Project

A **project** is the top-level unit in Autonomous — it represents one application or service group.

<Steps>

### Click Create Project

From your dashboard, click **"Create Project"**.

### Enter Project Details

Enter a project name. Description is optional.

### Choose Your Source

Select one of:
- **Connect a GitHub or GitLab repository** — Autonomous registers a webhook automatically
- **Upload a static archive** — for projects without a remote Git repository

### Project Created

The project record is created in the platform database. You're ready to configure environments and deploy.

</Steps>

<Callout type="warning">
  Only the project **owner** can modify or delete a project. Deleting a project cascades and removes all associated deployments, configurations, and secrets.
</Callout>

## Environment Configuration

Each project supports **three isolated environments** — each with its own environment variables, config files, and Kubernetes ConfigMaps applied independently at deploy time.

| Environment | Purpose |
|---|---|
| **dev** | Fast iteration, relaxed resource limits, verbose logging |
| **staging** | Production-mirror for QA and integration testing |
| **production** | Hardened config, auto-scaling enabled, alerts active |

**To configure an environment:**

1. Open your project and navigate to the **Environments** tab
2. Select the target environment (`dev` / `staging` / `prod`)
3. Add environment variables and config values
4. Save — configuration is stored per environment in the database
5. On the next deployment, config is applied as a Kubernetes **ConfigMap** in the appropriate namespace

## Secret Setup

Sensitive values — API keys, database passwords, tokens — are stored in **HashiCorp Vault**. They are never stored in the platform database, never committed to Git, and never appear in application logs.

<Steps>

### Open Secrets Tab

Navigate to your project's **Secrets** tab.

### Add a Secret

Enter the secret name and value.

### Stored in Vault

The value is written to HashiCorp Vault's KV engine, scoped to the project. The secret value is **never** stored in the platform database.

### Injected at Deploy Time

Vault injects the secret into the pod as an environment variable or mounted file. The application reads the value at runtime — no plaintext anywhere in the pipeline.

</Steps>

<Callout type="info">
  To rotate a secret, simply update its value in the Secrets tab. The new value will be injected on the next deployment — no code changes required.
</Callout>
